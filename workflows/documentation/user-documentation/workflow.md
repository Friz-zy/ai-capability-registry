# User Documentation Workflow

Workflow id: `user-documentation`

Use this workflow to create help center articles, onboarding guides, FAQ, how-to guides, and release notes for users.

## Files

- Manifest: `workflows/documentation/user-documentation/workflow.yaml`
- Common schemas: `workflows/templates/schemas/`
- Common artifact templates: `workflows/templates/artifacts/`

## Coordinator Notes

This workflow targets end users and customer-facing teams. Use `technical-documentation` for developer or operator documentation.

Follow the manifest for stage order, role delegation, gates, and acceptance criteria.
