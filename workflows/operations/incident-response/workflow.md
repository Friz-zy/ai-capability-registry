# Incident Response Workflow

Workflow id: `incident-response`

Use this workflow to coordinate active incidents, assess impact, mitigate, communicate, recover, and create follow-up actions.

## Files

- Manifest: `workflows/operations/incident-response/workflow.yaml`
- Common schemas: `workflows/templates/schemas/`
- Common artifact templates: `workflows/templates/artifacts/`

## Coordinator Notes

This workflow is for active or recent operational incidents. Use `postmortem` after stabilization when the main work is learning and prevention.

Follow the manifest for stage order, role delegation, gates, and acceptance criteria.
