# API Design Workflow

Workflow id: `api-design`

Use this workflow to design an API contract, authentication model, error model, pagination, versioning, and consumer documentation.

## Files

- Manifest: `workflows/engineering/api-design/workflow.yaml`
- Common schemas: `workflows/templates/schemas/`
- Common artifact templates: `workflows/templates/artifacts/`

## Coordinator Notes

This workflow should align providers and consumers before implementation. Use `integration-development` when the dominant work is connecting to an external system.

Follow the manifest for stage order, role delegation, gates, and acceptance criteria.
